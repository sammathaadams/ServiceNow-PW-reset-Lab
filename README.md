# ServiceNow-PW-reset-Lab
This lab uses a ServiceNow Developer Instance to simulate a user password reset workflow integrated with Active Directory. The goal is to practice handling common IT help desk requests, account verification, and secure password reset procedures in a controlled environment.

## Objective

## Tools Used
- ServiceNow Developer Instance
- Active Directory (Virtual Lab Environment)
- Windows Server / Windows Client VM

---

## Process Walkthrough

### 1. User Submits Password Reset Request
A password reset request is created in ServiceNow, simulating a user who is unable to access their account due to a forgotten or expired password.

---

### 2. Incident Review and Verification
The ticket is reviewed to confirm:
- User identity
- Account status
- Scope and priority of the request

Relevant details are documented in the incident record.

---

### 3. Active Directory Password Reset
Using Active Directory Users and Computers:
- The affected user account is located
- The password is reset according to security policy
- Options such as “User must change password at next logon” are applied if required

---

### 4. User Communication
The user is notified that their password has been reset and provided with next-step instructions, following secure communication guidelines.

---

### 5. Documentation and Ticket Closure
The ServiceNow ticket is updated with:
- Actions taken
- Resolution details
- Confirmation of restored access

The incident is then resolved and closed.
